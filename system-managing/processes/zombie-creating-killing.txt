Напрямую в bash создать зомби-процесс не получится, так как bash автоматически обрабатывает завершение дочерних процессов. Однако, можно использовать вызов системной функции C из bash, чтобы создать зомби-процесс. 

Вот пример скрипта на bash, который использует вызов системной функции C для создания зомби-процесса:

# Создание C-программы, которая создает зомби-процесс
echo '#include <stdlib.h>
#include <sys/types.h>
#include <unistd.h>

int main (){
   pid_t child_pid;
   child_pid = fork ();
   if (child_pid > 0) {
      sleep(60);
   }
   else {
      exit (0);
   }
   return 0;
}' > zombie.c

# Компиляция C-программы
gcc zombie.c -o zombie

# Запуск программы, которая создает зомби-процесс
./zombie &

# Удаление ненужных файлов
rm zombie.c zombie

# Проверка наличия зомби-процесса
ps aux | grep 'z'


Этот скрипт создает временный файл C-программы, компилирует его, запускает получившийся исполняемый файл и затем удаляет временные файлы. В результате выполнения этого скрипта будет создан зомби-процесс.

-------------------------------------------------------------------
Убить зомби-процесс напрямую невозможно, так как он уже завершил свою работу. Однако, вы можете убить родительский процесс, что заставит систему очистить все зомби-процессы, связанные с этим родительским процессом.

Вот основные шаги:

1. Сначала найдите PID (идентификатор процесса) зомби-процесса. Вы можете сделать это с помощью команды ps aux | grep 'z'. PID будет вторым столбцом в выводе.

2. Затем найдите PID родительского процесса (PPID). Это третий столбец в выводе.

3. Теперь убейте родительский процесс с помощью команды kill -9 PPID. Замените "PPID" на идентификатор родительского процесса, который вы нашли на предыдущем шаге.

4. Если родительский процесс не может быть убит, вы можете попытаться перезагрузить систему или воспользоваться командой kill -HUP 1, которая отправит сигнал SIGHUP инициализации системы, что заставит ее перезапустить все процессы.

Помните, что убийство процессов может привести к нежелательным последствиям, если вы не знаете, что делаете. Всегда убедитесь, что процесс, который вы собираетесь убить, действительно является тем, который вам нужен.
